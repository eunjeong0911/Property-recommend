## 🌳 공원온도 계산 시스템

### 📌 공원온도란?
부동산 매물의 **공원 환경**을 **30~43°C 온도**로 표현한 지표
- **30°C (차가움)** = 공원 부족 (가까운 공원 없음/작음/시설 부족)
- **36.5°C (평균 체온)** = 보통 수준 (일반 서울 주거지)
- **43°C (뜨거움)** = 공원 환경 우수 (가까운 공원 많음/크고 시설 좋음)

---

## 1단계: 공원 품질 점수 계산 (0~1점)

### 🏞️ 공원 점수 = 크기 + 시설 + 유형

각 공원의 품질을 3가지 요소로 평가합니다.

---

#### A. 크기 점수 (40% 가중치)
**공원면적이 얼마나 큰가?**

```
공원면적      정규화 점수    의미
─────────────────────────────────────
50 m²        0.000        🔴 초소형 공원
5,000 m²     0.100        🟠 작은 공원
50,000 m²    0.500        🟡 중간 크기
500,000 m²   0.900        🟢 큰 공원
6,691,885 m² 1.000        🟢 초대형 공원 (올림픽공원 등)
```

**정규화 방법:**
- 가장 작은 공원: 0.0점
- 가장 큰 공원: 1.0점
- 중간 공원: 면적에 비례하여 0~1 사이 값

**예시:**
```python
# 서울 전체 공원 면적 범위
최소: 48.8 m²
최대: 6,691,885 m² (올림픽공원)

# 어떤 공원의 면적이 10,000 m²라면
size_score = (10000 - 48.8) / (6691885 - 48.8) = 0.0015
```

---

#### B. 시설 풍부도 점수 (40% 가중치)
**공원에 어떤 시설들이 있는가?**

**5가지 시설 유형:**
1. **운동시설**: 농구장, 배드민턴장, 운동기구 등
2. **유희시설**: 그네, 시소, 조합놀이대, 미끄럼틀 등
3. **편익시설**: 화장실, 음수대, 파고라, 정자, 벤치 등
4. **교양시설**: 도서관, 전시관, 야외공연장 등
5. **기타시설**: 관리사무소, 주차장 등

**채점 방법:**
```
시설 개수    점수     의미
─────────────────────────────
0개         0.0점   🔴 시설 없음
1개         0.2점   🟠 최소 시설
2개         0.4점   🟠 기본 시설
3개         0.6점   🟡 보통
4개         0.8점   🟢 풍부
5개         1.0점   🟢 매우 풍부
```

**예시:**
```python
# 신사근린공원
시설(운동시설): 운동기구 등 → 1개 ✓
시설(유희시설): (없음) → 0개
시설(편익시설): 음수대, 정자, 장의자 등 → 1개 ✓
시설(교양시설): (없음) → 0개
시설(기타시설): (없음) → 0개

facility_count = 2개
facility_score = 2/5 = 0.4점
```

---

#### C. 공원 유형 점수 (20% 가중치)
**어떤 종류의 공원인가?**

```
공원 유형      점수    설명
──────────────────────────────────────
근린공원      1.0점   대규모 공원, 다양한 시설, 주민 쉼터
어린이공원    0.8점   어린이 전용, 놀이터 중심
소공원        0.7점   작은 규모, 간단한 휴게 시설
기타          0.9점   문화공원, 체육공원, 수변공원 등
```

**공원 유형 분포 (서울 전체):**
- 어린이공원: 1,012개 (59%)
- 근린공원: 332개 (19%)
- 소공원: 263개 (15%)
- 기타: 108개 (문화공원, 체육공원 등)

---

#### D. 공원 최종 점수 (S_park)

```
S_park = 0.4 × size_score + 0.4 × facility_score + 0.2 × type_score
```

**계산 예시 - 신사근린공원:**
```python
공원면적: 6,800 m²
  → size_score = 0.001 (작은 공원)

시설 개수: 2개
  → facility_score = 0.4 (기본 시설)

공원 유형: 근린공원
  → type_score = 1.0 (최고 등급)

S_park = 0.4×0.001 + 0.4×0.4 + 0.2×1.0
       = 0.0004 + 0.16 + 0.2
       = 0.360점
```

## 2단계: 거리 점수 계산

### 📏 거리 점수 = exp(-거리/400)

**얼마나 가까운 공원인가?**

```
거리 점수 공식: distance_score(d) = e^(-d/400)
```

**거리별 점수:**
```
거리          점수     의미           도보시간
───────────────────────────────────────────
0m           1.000    🟢 바로 옆      0분
100m         0.779    🟢 매우 가까움  1-2분
200m         0.607    🟢 가까움       2-3분
400m         0.368    🟡 적당함       5분
600m         0.223    🟠 조금 멀음    7-8분
800m         0.135    🟠 멀음         10분
1000m        0.082    🔴 매우 멀음    12-15분
```

**설계 의도:**
- **400m = 기준점** (도보 5분)
  - 일반 성인의 편안한 도보 거리
- **800m = 한계점** (도보 10분)
  - 이 이상은 점수가 급격히 감소
- **지수 감소 곡선**
  - 거리가 멀어질수록 점수가 급격히 낮아짐
  - 가까운 공원의 중요도 강조

## 3단계: 매물별 공원 점수 계산

### 🏠 매물의 공원 환경 = 주변 공원들의 거리 가중 평균

**계산 프로세스:**

```
1. 매물에서 모든 공원까지 거리 계산
   ↓
2. 반경 800m 이내 공원 선택
   (만약 없으면 가장 가까운 공원 1개)
   ↓
3. 각 공원의 점수 계산
   = 공원 품질(S_park) × 거리 점수
   ↓
4. 거리 가중 평균
   = Σ(점수 × 가중치) / Σ(가중치)
```

**거리 가중치:**
```
w = 1 / (1 + 거리/300)

거리        가중치    의미
───────────────────────────
0m          1.000    최대 가중치
150m        0.667    높은 가중치
300m        0.500    중간 가중치
600m        0.333    낮은 가중치
```

**계산 예시:**

```python
# 매물 A: 양재동 어느 위치

공원 1: 대청근린공원
  - 거리: 200m
  - S_park: 0.441
  - 거리점수: 0.607
  - 가중치: 0.6
  - 종합: 0.441 × 0.607 = 0.268

공원 2: 늘푸른근린공원
  - 거리: 500m
  - S_park: 0.441
  - 거리점수: 0.287
  - 가중치: 0.375
  - 종합: 0.441 × 0.287 = 0.127

공원 3: 개포서근린공원
  - 거리: 800m
  - S_park: 0.361
  - 거리점수: 0.135
  - 가중치: 0.273
  - 종합: 0.361 × 0.135 = 0.049

# 최종 점수 (거리 가중 평균)
S_park_property = (0.268×0.6 + 0.127×0.375 + 0.049×0.273) / (0.6+0.375+0.273)
                = 0.222 / 1.248
                = 0.178점
```

## 4단계: 온도로 변환 (30~43°C)

### 🌡️ 왜 온도로 표현하는가?

- **직관적**: "온도"라는 친숙한 개념으로 이해 쉬움
- **범위**: 30°C(추움) ~ 43°C(더움) = 13도 범위
- **평균**: 36.5°C = 사람 체온 = "딱 적당한" 느낌

```
공원온도 = 30 + (공원 점수 × 13)
```

**온도별 의미:**

| 온도 범위 | 공원 점수 | 의미 | 예시 |
|-----------|-----------|------|------|
| **30-32°C** 🥶 | 0.0-0.15 | 매우 부족 | 공원 없음/매우 멀음 |
| **32-34°C** 😰 | 0.15-0.31 | 부족 | 작은 공원만 있음 |
| **34-36°C** 😐 | 0.31-0.46 | 보통 | 일반 주거지 |
| **36-38°C** 🙂 | 0.46-0.62 | 양호 | 가까운 공원 있음 |
| **38-40°C** 😊 | 0.62-0.77 | 우수 | 좋은 공원 가까이 |
| **40-43°C** 🔥 | 0.77-1.0 | 최우수 | 대형 공원 바로 옆 |

**계산 예시:**
```python
# 앞의 예시 계속 (S_park_property = 0.178)

# 정규화 (6개 매물 중 상대적 위치)
# 최소: 0.065, 최대: 0.081
S_park_norm = (0.178 - 0.065) / (0.081 - 0.065) = 0.72

# 온도 변환
ParkTemp = 30 + (0.72 × 13)
        = 30 + 9.36
        = 39.36°C  🔥 우수 등급!
```

## ⚙️ 가중치 설정 근거

**공원 품질 점수:**
- 크기 40%: 넓은 공간이 중요 (산책, 운동, 휴식)
- 시설 40%: 다양한 활동 가능성
- 유형 20%: 공원 종류에 따른 기본 차이

**거리 점수:**
- exp(-d/400): 도보 5분 기준, 멀어질수록 급감
- 800m 반경: 도보 10분 한계

**거리 가중치:**
- 1/(1+d/300): 가까운 공원에 높은 가중치
- 멀리 있는 큰 공원보다 가까운 작은 공원이 실생활에 더 유용
