## 📊 편의시설온도 계산 결과

---

### 💡 편의시설온도란?

부동산 매물의 **생활 편의성**을 **30~43°C 온도**로 표현한 지표

**핵심 개념:**
- **필요도(Need)**: 매물 자체 시설이 부족할수록 외부 편의시설 의존도 증가
- **접근성(Access)**: 주변 편의시설/병원/약국까지의 거리 + 밀도
- **최종 점수**: Need × Access = 같은 위치라도 매물 옵션에 따라 다른 점수

---

### 🔢 계산 로직 상세

#### 1️⃣ 필요도(Need) 계산

매물의 생활시설을 분석하여 외부 편의시설 의존도 계산

**편의시설 필요도 (Need_conv):**

```
기본값: 0.7

+ 세탁기 없음: +0.1
  → 세탁소/빨래방 의존도 증가

+ 부실 주방: +0.1
  (냉장고 없음 OR (싱크대 없음 AND 가스/인덕션 없음))
  → 식당/편의점 의존도 증가

최대값: 1.0
```

**병원/약국 필요도:**
- 고정값 0.7 (매물 옵션과 무관)

**예시:**
```
매물 A (풀옵션): 세탁기○, 냉장고○, 싱크대○, 가스레인지○
  → Need_conv = 0.7 (낮음)

매물 B (옵션부족): 세탁기✗, 냉장고✗
  → Need_conv = 0.9 (높음, 외부 시설 필수)
```

---

#### 2️⃣ 접근성(Access) 계산

POI까지의 **거리**와 주변 **밀도**를 반영합니다.

**공식:**
```
Access = 0.6 × 거리점수 + 0.4 × 밀도점수
```

**파라미터:**

| 유형 | 거리 기준 (d_max) | 밀도 반경 | 포화 개수 (n_sat) |
|------|------------------|----------|------------------|
| **편의시설** | 800m (도보 10분) | 500m | 5개 |
| **병원** | 1200m (도보 15분) | 800m | 2개 |
| **약국** | 1200m (도보 15분) | 800m | 2개 |

**거리 점수 계산:**
```
distance_score = max(0, 1 - d/d_max)

예시 (편의시설, d_max=800m):
- d = 0m   → 1.0 (바로 앞)
- d = 400m → 0.5 (중간)
- d = 800m → 0.0 (경계)
- d > 800m → 0.0 (너무 멀음)
```

**밀도 점수 계산:**
```
density_score = min(n/n_sat, 1.0)

예시 (편의시설, n_sat=5):
- n = 0개 → 0.0 (없음)
- n = 2개 → 0.4 (부족)
- n = 5개 → 1.0 (충분, 만점)
- n = 10개 → 1.0 (과잉, 만점 유지)
```

**왜 거리 60%, 밀도 40%?**
- **거리 중심(60%)**: 가장 가까운 POI가 중요 (긴급 상황, 편리성)
- **밀도 반영(40%)**: 선택의 다양성 (여러 편의점, 식당 중 선택 가능)

---

#### 3️⃣ 유형별 점수 계산

**공식:**
```
S_type = Need × Access
```

**예시:**
```
편의시설 점수:
  Need_conv = 0.9 (세탁기, 주방 부족)
  Access_conv = 0.8 (편의시설 가까움)
  → S_conv = 0.9 × 0.8 = 0.72

병원 점수:
  Need_med = 0.7 (고정)
  Access_med = 0.6 (병원 보통)
  → S_med = 0.7 × 0.6 = 0.42

약국 점수:
  Need_pharm = 0.7 (고정)
  Access_pharm = 0.6 (약국 보통)
  → S_pharm = 0.7 × 0.6 = 0.42
```

---

#### 4️⃣ 최종 총점 및 온도 변환

**총점 계산 (가중 평균):**
```
S_amenity = 0.5 × S_conv + 0.25 × S_med + 0.25 × S_pharm
```

**가중치 근거:**
- **편의시설 50%**: 일상 생활에서 가장 자주 이용 (매일)
- **병원 25%**: 중요하지만 자주 이용하지 않음 (가끔)
- **약국 25%**: 필요할 때 중요 (가끔)

**온도 변환:**
```
AmenityTemp = 30 + 13 × S_amenity
```

**온도 의미:**
| S_amenity | 온도 | 의미 |
|-----------|------|------|
| 0.0 | 30.0°C | 최저 (편의시설 접근 매우 부족) |
| 0.5 | 36.5°C | 평균 체온 (보통 수준) |
| 0.7 | 39.1°C | 양호 (편의 환경 좋음) |
| 1.0 | 43.0°C | 최고 (편의시설 환경 우수) |

### 📈 결과 해석 가이드

#### 온도 구간별 의미

**🟦 낮음 (30-34°C)**
- **특징**: 매물 옵션 부족 + 주변 편의시설 적음
- **상황**: 세탁소, 식당 등 외부 시설 필요하지만 접근 어려움
- **평가**: 생활 편의성 낮음
- **예시**: 세탁기 없고, 가장 가까운 편의점이 1km 이상

**🟨 보통 (34-39°C)**
- **특징**: 일반적인 주거 환경
- **상황**: 매물 옵션 또는 주변 편의시설 중 하나는 양호
- **평가**: 기본적인 생활 가능
- **예시**: 풀옵션이지만 편의시설 거리 중간, 또는 옵션 부족하지만 편의시설 가까움

**🟥 높음 (39-43°C)**
- **특징**: 풀옵션 매물 + 주변 편의시설 풍부
- **상황**: 편의점, 식당, 병원, 약국 모두 가까움
- **평가**: 생활 편의성 매우 우수
- **예시**: 세탁기○, 주방○, 도보 5분 내 편의점 5개, 병원, 약국

---

### 🎯 실전 활용 팁

1. **매물 선택 시:**
   - 편의시설온도 38°C 이상 → 생활 편의성 보장
   - 37°C 이하 → 외부 시설 접근 확인 필요

2. **옵션과 위치 트레이드오프:**
   - 풀옵션(Need↓) + 외곽(Access↓) vs 옵션부족(Need↑) + 역세권(Access↑)
   - Need × Access 곱이므로 균형이 중요

3. **다른 온도와 종합 판단:**
   - 교통온도(출퇴근) + 공원온도(환경) + 편의시설온도(생활)
   - 3가지 온도를 함께 고려하여 최적의 매물 선택
