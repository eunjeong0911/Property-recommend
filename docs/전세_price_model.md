# 전세 평당 전세금 예측 모델

서울 전세 매물의 **평당 전세금**을 예측하는 머신러닝 모델입니다.

## 📌 모델 개요

이 모델은 전세 매물 정보를 입력받아 **평당 전세금(만원)**을 예측합니다.

### 예시
- **입력**: 강남구 25평 아파트, 5층, 방 3개, 욕실 1개, 관리비 10만원...
- **출력**: 평당 전세금 약 2,000만원
- **계산**: 총 전세금 = 평당 전세금 × 전용면적(평)

### 계산 공식
```
평당 전세금 = 전세금 / 전용면적(평)
```

## 📈 모델 성능 지표

모델의 성능은 다음 지표로 평가됩니다:

| 지표 | 설명 | 좋은 값 |
|------|------|--------|
| **MAE** | 평균 절대 오차 (만원) | 낮을수록 좋음 |
| **RMSE** | 평균 제곱근 오차 (만원) | 낮을수록 좋음 |
| **R²** | 결정계수 (설명력) | 1에 가까울수록 좋음 |
| **MAPE** | 평균 절대 백분율 오차 (%) | 낮을수록 좋음 |

### 기대 성능 (참고)
```
MAE  : 약 300만원   ← 평균적으로 300만원 오차
RMSE : 약 480만원
R²   : 약 0.78      ← 78% 설명력
MAPE : 약 15%       ← 평균 15% 오차율
```

## ❓ FAQ

### Q1. 로그 변환 이유

**A.** 평당가의 분포가 한쪽으로 치우쳐 있어서(오른쪽 꼬리가 긴 분포), 로그 변환을 통해 정규분포에 가깝게 만들면 모델 성능이 향상됩니다.

```python
# 로그 변환
y_train_log = np.log1p(y_train)  # log(x + 1)

# 예측 후 역변환
y_pred = np.expm1(y_pred_log)  # exp(x) - 1
```

### Q2. Target Encoding이란?

**A.** 구/동 같은 카테고리 변수를 숫자로 바꾸는 기법입니다. 각 구/동의 평균 평당가를 사용합니다.

- 예: 강남구 → 2,000만원 (강남구 평균 평당가)
- 예: 마포구 → 1,500만원 (마포구 평균 평당가)

**과적합 방지**: Leave-One-Out 방식으로 자기 자신을 제외한 평균을 사용합니다.
