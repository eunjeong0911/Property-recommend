# 중개사 신뢰도 평가 모델 (Trust Model)

## 🎯 이 모델이 하는 일

부동산 중개사의 **신뢰도 온도(0~1 점수)**와 **등급(D, C, B, A, S)**을 자동으로 계산하는 머신러닝 파이프라인입니다.

**핵심 아이디어:**
1. 도메인 지식 기반의 **룰 스코어** (거래 실적, 보증보험 등)
2. 머신러닝 기반의 **분류 확률** (RandomForest)
3. 두 점수를 **50:50으로 앙상블**하여 최종 신뢰도 산출

### 1단계: 데이터 로드 (`_00_load_data.py`)

**목적:** 서울시 중개사 데이터 불러오기

```python
df = load_data("data/seoul_broker_clean.csv")
```

**입력 데이터:**
- 380개 중개사 정보
- 주요 컬럼: `거래완료`, `등록매물`, `총매물수`, `registDe`(등록일), `estbsEndDe`(보증보험 만료일), `ldCodeNm`(지역명) 등

---

### 2단계: 룰 기반 스코어링 (`_01_rule_score.py`)

**목적:** 도메인 지식을 활용한 기본 점수 계산

#### 생성되는 파생 변수

| 변수명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `영업일수` | 등록일 ~ 현재 | 중개사 운영 기간 |
| `일평균거래` | 거래완료 ÷ 영업일수 | 하루 평균 거래 건수 |
| `보증보험유효` | 만료일 >= 현재 | 보증보험 유효 여부 (0 or 1) |
| `거래성사율` | 거래완료 ÷ 총매물수 | 매물 대비 실제 거래 비율 |
| `지역권평균성사율` | 지역별 평균 성사율 | 해당 지역의 평균 성과 |

#### Rule Score 계산 공식

```python
rule_score = (거래성사율 × 40) + (일평균거래 × 30) + (보증보험유효 × 20) + (지역권평균성사율 × 10)
```

**가중치 설계 이유:**
- **거래성사율 40%**: 실제 거래 능력이 가장 중요
- **일평균거래 30%**: 활동성과 경험
- **보증보험유효 20%**: 법적 안전성
- **지역권평균성사율 10%**: 지역 시장 환경 반영

#### 등급 분류 (grade)

`rule_score`를 기준으로 **Quantile 방식**으로 5등급 분류:
- D (하위 20%) → C (20~40%) → B (40~60%) → A (60~80%) → S (상위 20%)

---

### 3단계: 피처 엔지니어링 (`_02_feature_engineering.py`)

**목적:** 머신러닝 모델을 위한 추가 피처 생성

#### 기본 피처 (2개)

| 피처명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `등록비율` | 등록매물 ÷ 총매물수 | 활발한 매물 등록 비율 |
| `규모지수` | log(총매물수 + 1) | 중개사 규모 (로그 스케일) |

#### 지역 기반 피처 (4개)

| 피처명 | 계산 방법 | 의미 |
|--------|-----------|------|
| `지역내순위` | 지역별 거래완료 순위 | 해당 지역에서의 경쟁력 |
| `지역내백분위` | 지역별 거래완료 백분위 | 지역 내 상위 몇 % (0~1) |
| `지역중개사수` | 지역별 중개사 개수 | 경쟁 강도 |
| `지역평균대비거래비율` | 거래완료 ÷ 지역평균 | 지역 평균 대비 성과 |

**왜 지역 피처가 중요한가?**
- 강남구와 도봉구는 시장 환경이 다름
- 같은 거래 건수라도 지역 내 경쟁 상황에 따라 의미가 달라짐

---

### 4단계: 분류 모델 학습 (`_05_classification_model.py`)

**목적:** RandomForest로 등급(D~S) 예측 모델 학습

#### 모델 설정

```python
RandomForestClassifier(
    n_estimators=50,         # 트리 50개 (과적합 방지)
    max_depth=6,             # 최대 깊이 6 (단순화)
    min_samples_split=20,    # 분할 최소 샘플 20개
    min_samples_leaf=10,     # 리프 최소 샘플 10개
    class_weight='balanced', # 클래스 불균형 처리
    random_state=42
)
```

**일반화 우선 설계:**
- 트리 개수와 깊이를 제한하여 과적합 방지
- 작은 데이터셋(380개)에 최적화

#### 사용 피처 (10개)

**원본 데이터 (5개):**
- `거래완료`, `등록매물`, `총매물수`, `영업일수`, `보증보험유효`

**파생 피처 (5개):**
- `등록비율`, `규모지수` (기본)
- `지역권평균성사율`, `지역내백분위`, `지역중개사수` (지역)

**제외된 피처 (데이터 누수 방지):**
- ❌ `거래성사율`: 타겟(grade)과 직접 연관
- ❌ `일평균거래`: 거래완료/영업일수 = 간접 누수
- ❌ `지역내순위`: 백분위로 대체 (더 일반화됨)

#### 데이터 전처리

```python
# RobustScaler 사용 (이상치에 강함)
scaler = RobustScaler()
X_train_scaled = scaler.fit_transform(X_train)
```

**Train/Test Split:**
- 80% 학습 (304개) / 20% 테스트 (76개)
- Stratified 방식 (각 등급 비율 유지)

#### 출력 결과

| 컬럼명 | 설명 |
|--------|------|
| `clf_pred` | 모델이 예측한 등급 (D, C, B, A, S) |
| `clf_score` | S등급일 확률 (0~1) - 신뢰도 점수로 사용 |

#### 모델 성능


### 5단계: 앙상블 & 최종 등급 (`_06_ensemble.py`)

**목적:** 룰 점수와 ML 확률을 결합하여 최종 신뢰도 산출

#### 앙상블 공식

```python
# 1. rule_score를 0~1로 정규화
rule_norm = (rule_score - min) / (max - min)

# 2. 앙상블 (50:50)
final_temperature = (rule_norm × 0.5) + (clf_score × 0.5)
```

**왜 50:50인가?**
- 도메인 지식(룰)과 데이터 패턴(ML)의 균형
- 한쪽에 과도하게 의존하지 않음

#### 최종 등급 분류

```python
final_grade = pd.qcut(final_temperature, q=5, labels=["D", "C", "B", "A", "S"])
```

- Quantile 방식으로 각 등급 20%씩 균등 분포

## 🎯 모델 설계 철학

### 1. 데이터 누수 방지

**제외된 피처:**
- `거래성사율`: 타겟(grade)과 직접 연관
- `일평균거래`: 거래완료를 이미 사용하므로 중복

**결과:** 모델이 실제 패턴을 학습하도록 유도

### 2. 일반화 우선

**과적합 방지 전략:**
- 트리 개수 감소 (100 → 50)
- 최대 깊이 제한 (8 → 6)
- 최소 샘플 수 증가 (split 20, leaf 10)

**결과:** 과적합 차이 2.3% (매우 건강)

### 3. 지역 특성 반영

**지역 기반 피처 4개:**
- 같은 거래 건수라도 지역 환경에 따라 다르게 평가
- 강남구 10건 vs 도봉구 10건의 의미 차이 반영

### 4. 클래스 균형

**Quantile 기반 등급:**
- 각 등급이 정확히 20%씩 분포
- 상대 평가 방식 (절대 점수가 아님)

### 5. 앙상블 균형

**50:50 비율:**
- 도메인 지식(룰)과 데이터 패턴(ML)의 조화
- 한쪽에 편향되지 않음

### 신규 중개사 신뢰도 예측

```python
import pickle
import pandas as pd

# 모델 로드
with open("models/trust_model.pkl", "rb") as f:
    package = pickle.load(f)

model = package['model']
scaler = package['scaler']

# 신규 데이터 준비 (10개 피처 필요)
new_data = pd.DataFrame({
    '거래완료': [15],
    '등록매물': [30],
    '총매물수': [50],
    '영업일수': [365],
    '보증보험유효': [1],
```

## ❓ FAQ

**Q1. 왜 RandomForest를 선택했나요?**
- 작은 데이터셋(380개)에 적합
- 피처 중요도 해석 가능
- 과적합 제어 용이

**Q2. 왜 등급이 Quantile 방식인가요?**
- 상대 평가 방식 (상위 20%가 S등급)
- 시간이 지나도 등급 분포 유지
- 절대 점수는 시장 상황에 따라 변동 가능

**Q3. 신규 중개사는 어떻게 평가하나요?**
- 영업일수가 짧으면 `일평균거래`가 불안정
- 최소 3개월 이상 데이터 권장
- 초기에는 `rule_score` 비중을 높일 수 있음

**Q4. 모델 재학습 주기는?**
- 분기별 1회 권장
- 신규 중개사 추가 시
- 시장 환경 변화 시 (예: 부동산 정책 변경)
