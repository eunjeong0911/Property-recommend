# Trust Score Key Columns Explanation

사용자가 질문하신 두 가지 핵심 변수(`보정된_기준값`, `베이지안_성사율`)에 대해, **실제 출력된 "용산구" 데이터**를 예시로 들어 정확하게 설명해 드립니다.

---

## 1. 보정된 기준값 (Adjusted Prior)
> **한 줄 요약**: "이 지역(용산구)에서 활동한다면, **기본적으로 기대되는 평균 점수**입니다."

*   **값**: `0.77931` (용산구의 모든 중개사에게 **동일하게** 적용됨)
*   **의미**: 
    *   개별 중개사의 실력을 보보기 전에, **"용산구라는 시장 자체가 얼마나 잘 팔리는 곳인가?"**를 먼저 계산한 값입니다.
    *   용산구는 거래가 활발하고 성사율이 좋은 지역이므로, 전체 평균(0.53)보다 높은 **0.78점**이라는 높은 **출발점(Base Line)**을 가집니다.
    *   즉, "용산구 중개사라면 못해도 78점 정도는 하겠구나"라고 미리 깔고 들어가는 점수입니다.

---

## 2. 베이지안 성사율 (Bayesian Success Rate)
> **한 줄 요약**: "지역 기대치(78점)와 내 진짜 실력(Raw Rate)을 **데이터 개수에 따라 섞은 최종 점수**입니다."

베이지안 로직은 **"데이터(매물)가 적으면 지역 평균을 믿고, 데이터가 많으면 내 실력을 믿는다"**는 원칙으로 작동합니다.
(※ 현재 임계값 $m = 91$개)

### 실제 데이터 예시 풀이

#### **사례 1: 데이터가 적은 중개사 (Index 1)**
*   **상황**: 매물 **81개** (임계값 91개보다 적음 / 데이터 부족)
*   **단순 실력**: 76건 완료 $\rightarrow$ 성공률 **93.8%** (와! 엄청 잘한다?)
*   **베이지안 판단**: "매물이 81개밖에 없네? 아직 운일 수도 있어. 지역 평균(78점) 쪽으로 좀 깎자."
*   **계산**: 지역 평균(0.78) 비중을 높여서 섞음.
*   **결과**: **`0.85 (-8.4% 하향 조정)`**
    *   94점짜리 실력이지만, 검증 데이터가 부족해 85점으로 **안전하게 보정**되었습니다.

#### **사례 2: 데이터가 충분한 중개사 (Index 0)**
*   **상황**: 매물 **171개** (임계값 91개보다 많음 / 데이터 충분)
*   **단순 실력**: 161건 완료 $\rightarrow$ 성공률 **94.1%** (여기도 잘한다!)
*   **베이지안 판단**: "매물을 171개나 팔았어? 이건 운이 아니라 진짜 실력이야. 네 점수 인정해줄게."
*   **계산**: 내 실력(0.94) 비중을 높여서 섞음.
*   **결과**: **`0.88 (-5.6% 조금만 하향)`**
    *   위 사례보다 매물이 많아서, 본인 실력을 더 많이 인정받아 점수가 덜 깎였습니다.

### 3. 요약 (Conclusion)
| 구분 | 보정된 기준값 (Prior) | 베이지안 성사율 (Final Score) |
| :--- | :--- | :--- |
| **성격** | **지역의 점수** (Context) | **나의 점수** (Skill) |
| **변화** | 같은 지역이면 모두 똑같음 | 중개사마다 다름 |
| **역할** | "너 신입이야? 그럼 일단 이 점수(78점)부터 시작해." | "네가 증명한 만큼 점수(93점)를 줄게, 못하면 78점으로 돌아와." |

결국 `베이지안_성사율`은 **"운빨"을 제거하고 진짜 실력을 찾아낸 최종 성적표**입니다.
