# 월세 가격 분류 모델 (ML 기반)


## 1. 문제 정의

### 1.1 목표
서울시 월세 실거래가 데이터를 활용하여, 특정 매물이 **저렴(UNDERPRICED)**, **적정(FAIR)**, **비쌈(OVERPRICED)** 중 어느 범주에 속하는지 분류하는 3중 분류 모델 개발

### 1.2 데이터
- **학습 데이터**: 2024년 8월 ~ 2025년 8월 월세 실거래가 (1년치)
- **테스트 데이터**: 2025년 9월 ~ 2025년 10월 월세 실거래가
- **데이터 출처**: 서울시 월세 실거래가 + 한국은행 금리 데이터

---

## 2. 타깃 변수 설계

### 2.1 가격 지표: 환산보증금 평당가

**이유:**

월세는 보증금과 월세라는 두 가지 요소로 구성되어 있어, 단순히 월세만으로는 가격을 비교하기 어려움

```
환산보증금(만원) = 보증금(만원) + (월세(만원) × 12) / 적용이자율
적용이자율 = (기준금리 + 2.0%) / 100
환산보증금_평당가 = 환산보증금(만원) / 전용평수
```

**추가 이유:**
1. **월세와 보증금의 통합**: 월세를 보증금으로 환산하여 단일 지표로 통합
2. **금리 반영**: 기준금리를 반영하여 시장 상황을 고려
3. **면적 정규화**: 평당가로 변환하여 면적에 따른 가격 차이를 제거
4. **비교 가능성**: 다양한 계약 조건(보증금 높음/월세 낮음 vs 보증금 낮음/월세 높음)을 동일 선상에서 비교 가능

### 2.2 분류 기준: 행정동×건물용도별 분위수

**왜 행정동×건물용도별 분위수를 사용했는가?**

```
33.3% 미만 → 저렴(0)
33.3% ~ 66.7% → 적정(1)
66.7% 초과 → 비쌈(2)
```


**선택 이유:**
1. **지역 특성 반영**: 강남구와 노원구의 가격 수준이 다르기 때문에, 전체 분위수가 아닌 자치구별 분위수를 사용
   - 예: 강남구의 "저렴"은 노원구의 "비쌈"보다 절대가격이 높을 수 있지만, 해당 지역 내에서는 상대적으로 저렴
2. **균형있는 클래스 분포**: 33.3%, 66.7% 분위수를 사용하여 클래스 불균형을 완화
3. **실용성**: 사용자에게 "이 지역에서 이 가격이 저렴/적정/비쌈" 정보를 제공하는 것이 더 유용

### 3.2 Feature Engineering 전략

#### 3.2.1 지역 정보 Feature

**왜 Label Encoding을 사용했는가?**

```python
자치구명_LE, 법정동명_LE, 자치구_건물용도_LE
```

**선택 이유:**
1. **Tree 모델 최적화**: XGBoost, LightGBM 같은 Tree 기반 모델은 Label Encoding이 효율적
   - One-Hot Encoding 시 차원이 폭발 (25개 자치구 + 400개 법정동 = 400+ 차원)
   - Label Encoding 시 3개 컬럼만으로 표현 가능
2. **순서 무관**: Tree 모델은 분할(split) 기반이므로 Label 숫자의 순서가 의미 없음
   - 강남구=0, 노원구=1이어도 모델이 자동으로 패턴 학습

**왜 복합 카테고리(자치구_건물용도)를 만들었는가?**

**선택 이유:**
1. **교호작용 포착**: 강남구의 아파트 vs 노원구의 아파트는 가격 패턴이 다름
2. **세밀한 분류**: 자치구와 건물용도를 독립적으로 사용하는 것보다 더 정확한 분류 가능

**왜 구_권역(동부/서부/남부/북부)을 만들었는가?**

**선택 이유:**
1. **일반화 성능 향상**: 자치구 수준보다 더 넓은 지역 패턴 학습
2. **Sparse Data 완화**: 일부 자치구는 데이터가 적을 수 있으므로, 권역으로 그룹화하여 안정적인 학습

#### 3.2.2 건물 특성 Feature

**왜 면적을 범주화(면적_qcat)했는가?**

**선택 이유:**
1. **비선형성 포착**: 면적과 가격의 관계는 선형이 아님
   - 초소형(20㎡): 1인 가구, 저렴
   - 대형(100㎡+): 고급 매물, 비쌈
2. **이상치 영향 감소**: 극단적인 면적 값의 영향을 줄임
3. **해석 용이성**: "소형", "중형" 같은 범주가 실제 부동산 시장의 분류와 일치

**왜 층을 구간화(층_bin)했는가?**

**선택 이유:**
1. **부동산 관행 반영**: 저층(1~3층), 중층(4~6층), 중고층(7~11층), 고층(12층+)은 실제 부동산 시장의 분류
2. **비선형성 포착**: 1층(습기, 소음) vs 5층(적당) vs 20층(뷰) 같은 비선형 관계 학습

**왜 건축연차와 건축시대를 모두 사용했는가?**

**선택 이유:**
1. **건축연차** (연속): 1년 단위의 세밀한 노후도 정보
2. **건축시대** (범주): 건축 기법, 설계 트렌드가 10년 단위로 변화 (예: 2000년 이전은 노후, 20년대 이후는 신축)

#### 3.2.3 시간 특성 Feature

**왜 계약_계절을 사용했는가?**

**선택 이유:**
1. **이사 수요 패턴**: 봄(3~5월), 여름(6~8월)은 이사 성수기로 가격 변동 가능
2. **비선형성**: 계약월(1~12)보다 계절(봄/여름/가을/겨울)이 더 의미 있는 패턴
3. **순환성**: 1월과 12월이 가까운 관계임을 모델이 학습 가능

**왜 분기, 반기는 제외했는가?**

**선택 이유:**
1. **중복성**: 계절과 비슷한 정보를 제공하므로 제거 (Feature 수 감소)
2. **단순화**: 모델 복잡도를 줄이기 위해 가장 의미 있는 Feature만 선택

#### 3.2.4 지역 집계 Feature

**왜 자치구_거래량_구간을 사용했는가?**

**선택 이유:**
1. **유동성 반영**: 거래량이 많은 지역은 시장이 활발하여 가격 형성이 합리적
2. **인기도 지표**: 거래량이 적은 지역은 특수한 가격 패턴 가능

**왜 자치구_용도_월별_임대료_평균을 사용했는가?**

**선택 이유:**
1. **지역×용도×시간 패턴**: 강남구 아파트의 2024년 1월 vs 2025년 8월 평균 임대료 변화 반영
2. **상대적 가격 평가**: 현재 매물이 해당 지역/용도/시점의 평균 대비 비싼지 저렴한지 판단 가능
3. **시계열 트렌드**: 월별 변화를 반영하여 계절성, 금리 변화 등의 영향 포착

**왜 법정동_용도_월별_임대료_평균도 사용했는가?**

**선택 이유:**
1. **더 세밀한 지역 정보**: 같은 자치구 내에서도 법정동별로 가격 차이 큼 (예: 강남구 내 삼성동 vs 도곡동)
2. **보완적 역할**: 자치구 수준과 법정동 수준의 정보를 모두 제공하여 모델이 선택 가능

**왜 자치구_월별_임대료수준_구간을 사용했는가?**

**선택 이유:**
1. **명시적 상대 비교**: 현재 매물이 해당 자치구×월 평균 대비 ±10% 기준으로 저렴/적정/비쌈 판단
2. **타깃과 직접적 연관**: 타깃 변수(저렴/적정/비쌈)와 유사한 개념으로, 모델 학습에 강력한 신호 제공

#### 3.2.5 금리 정보 Feature

**왜 6개의 금리를 모두 사용했는가?**

**선택 이유:**
1. **다양한 금융 상황 반영**:
   - **KORIBOR**: 은행 간 단기 자금 조달 금리 (시장 유동성 반영)
   - **기업대출**: 기업 자금 조달 비용 (경기 상황 반영)
   - **전세자금대출**: 전세 시장과 직접 연관
   - **CD**: 양도성예금증서 금리 (단기 금리 지표)
   - **무담보콜금리**: 초단기 금리 (중앙은행 정책 반영)
   - **변동형주택담보대출**: 주택 구매 비용 (주택 시장 전반 영향)
2. **상관관계 다양성**: 각 금리는 서로 다른 패턴을 가지며, Tree 모델은 자동으로 중요한 금리 선택 가능
3. **환산보증금 계산과 연관**: 환산보증금 계산 시 금리를 사용하므로, 원본 금리 정보도 중요

**왜 기준금리_레벨, 소비자물가_레벨을 범주화했는가?**

**선택 이유:**
1. **비선형 구간 효과**: 기준금리 2.5% vs 3.5%는 단순 0.5%p 차이가 아니라 "저금리 환경" vs "고금리 환경"이라는 질적 차이
2. **임계값 효과 포착**: 특정 금리 구간을 넘으면 시장 행동이 달라짐
3. **해석 용이성**: "고금리 시기의 강남구 아파트"처럼 직관적 해석 가능

#### 3.2.6 상대적 지표 Feature

**왜 자치구 수준 z-score를 사용했는가?**

```python
전용면적_자치구수준_z, 건축연도_자치구수준_z
```

**선택 이유:**
1. **지역 내 상대적 위치 파악**:
   - 강남구에서 60㎡는 작은 편이지만, 노원구에서는 중간 크기
   - 절대값이 아닌 "해당 지역에서 얼마나 큰/작은 매물인가" 정보 제공
2. **이상치 탐지**: z-score > 2 또는 < -2인 매물은 해당 지역에서 특이한 매물
3. **표준화**: z-score로 변환하여 자치구 간 비교 가능

**왜 층수_자치구수준_z는 제외했는가?**

**선택 이유:**
1. **중복성**: 층_bin과 층 정보가 이미 충분하므로 제거
2. **Feature 선택**: 실험 결과 성능 향상에 기여도가 낮아서 제외

## 5. 학습 및 평가 전략

### 5.1 시계열 기반 Train/Val/Test 분할

```
Train: 2024-01 ~ 2025-05 (학습)
Val:   2025-06 ~ 2025-08 (검증, Early Stopping)
Test:  2025-09 ~ 2025-10 (최종 평가)
```

**선택 이유:**
1. **데이터 특성**: 월세 실거래가는 시계열 데이터
2. **현실 반영**: 과거 데이터로 학습하여 미래 데이터 예측 (실제 사용 시나리오와 동일)
3. **Data Leakage 방지**: Random Split 시 미래 정보가 학습에 포함될 수 있음

**주의사항:**
- **절대로 K-Fold Cross Validation을 사용하지 않음**: 시계열 데이터에서는 미래 정보가 과거 학습에 포함되는 문제 발생

### 5.2 평가 지표

#### 주요 지표: F1-Macro

**왜 F1-Macro를 주요 지표로 사용했는가?**

**선택 이유:**
1. **클래스 불균형 고려**: Accuracy는 다수 클래스에 편향될 수 있음
   - 예: "모두 적정(1)로 예측"해도 Accuracy가 60%가 나올 수 있음
2. **모든 클래스 동등 중요**: 저렴(0), 적정(1), 비쌈(2) 모두 중요하므로 Macro Average 사용
   - Macro: 각 클래스 F1의 평균 (클래스 균등 가중치)
   - Weighted: 샘플 수에 비례한 가중 평균 (다수 클래스 편향)
3. **Precision과 Recall의 조화 평균**: 두 지표 모두 중요

#### 보조 지표

| 지표 | 의미 | 사용 이유 |
|------|------|----------|
| **Accuracy** | 전체 정확도 | 전반적인 성능 파악 |
| **F1-Weighted** | 샘플 수 가중 F1 | 다수 클래스 성능 확인 |
| **ROC-AUC (OVR)** | 확률 기반 평가 | 분류 신뢰도 평가 |
| **Confusion Matrix** | 클래스별 혼동 | 어떤 클래스를 잘못 예측하는지 파악 |
| **Classification Report** | 클래스별 Precision/Recall/F1 | 세부 성능 분석 |
